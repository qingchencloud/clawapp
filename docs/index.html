工作原理

SSE 事件流 + HTTP POST，Ed25519 设备签名自动握手

手机浏览器 Token 认证
SSE + POST
ClawApp Server :3210 代理 + 离线缓存
WebSocket
OpenClaw Gateway :18789 Ed25519 签名

为什么选择 ClawApp

对比 OpenClaw 原生 Web UI 的移动端体验

特性OpenClaw Web UIClawApp
移动端适配桌面端设计移动端优先
通信协议仅 WebSocketSSE + HTTP POST
锁屏 / 切后台连接断开SSE 自动恢复
离线消息不支持IndexedDB 持久化
断线续传需手动刷新Last-Event-ID 自动补发
语音输入不支持语音转文字
PWA 安装不支持添加到主屏幕
外网访问需自行配置cftunnel 一键穿透

快速开始

一键部署，打开浏览器即可使用

一键部署 (Mac / Linux)
curl -fsSL https://raw.githubusercontent.com/qingchencloud/clawapp/main/install.sh | bash
一键部署 (Windows PowerShell)
irm https://raw.githubusercontent.com/qingchencloud/clawapp/main/install.ps1 | iex

自动检测环境、安装依赖、构建前端、配置 Token，支持 PM2 常驻运行

git clone https://github.com/qingchencloud/clawapp.git cd clawapp # 创建 .env echo 'PROXY_TOKEN=my-secret-token' > .env echo 'OPENCLAW_GATEWAY_TOKEN=你的token' >> .env # 启动 docker compose up -d --build # 手机访问 http://你的电脑IP:3210
git clone https://github.com/qingchencloud/clawapp.git cd clawapp npm run install:all npm run build:h5 cp server/.env.example server/.env # 编辑 server/.env 填入 token npm start # 手机访问 http://你的电脑IP:3210

外网访问

不在同一 WiFi？用 cftunnel 一条命令搞定公网穿透

临时分享（零配置）

cftunnel quick 3210 # https://xxx-yyy.trycloudflare.com

免费、自动 HTTPS、无需域名和公网 IP

固定域名（持久稳定）

cftunnel init cftunnel create my-tunnel cftunnel add clawapp 3210 --domain chat.example.com cftunnel up

自有域名 + 开机自启 cftunnel install

cftunnel GitHub 产品主页

下载 & 使用

部署服务端后，选择适合你的方式连接

iPhone / iPad

Safari 打开即用，无需 App Store：

  1. Safari 打开 http://你的IP:3210
  2. 输入服务器地址和 Token，连接聊天
  3. 点击 Safari 底部分享按钮
  4. 选择「添加到主屏幕」

PWA 模式支持全屏 + 离线缓存

Android

两种方式可选：

浏览器直接用

Chrome 打开后会提示「添加到主屏幕」

下载 APK 安装

下载最新 APK

从 GitHub Releases 下载

电脑浏览器

直接在电脑浏览器中使用：

  1. 打开 http://localhost:3210
  2. 输入 Token 连接即可
